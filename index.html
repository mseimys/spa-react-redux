UI data binding is a software design pattern to simplify development of GUI applications. UI data binding binds UI elements to an application domain model. Most frameworks employ the Observer pattern as the underlying binding mechanism.

Functional Reactive Programming

What is it? The simple answer to this question is that there is no simple answer. FRP is about "datatypes that represent a value 'over time'" You can define them and combine them, pass them into & out of functions x = [[mouse-x]]; y = [[mouse-y]]; minX = x - 16; minY = y - 16; maxX = x + 16; maxY = y + 16; rectangle(minX, minY, maxX, maxY) * Immutable data --- [1,2] --> [1,2,3] - amazing change detection perfomance, new stream based on another stream does not modify the original stream in any way. This is a property called immutability * Pure functions * One-way state transitions * Observables (subscribing to events) * Static typing (avoids runtime errors)
